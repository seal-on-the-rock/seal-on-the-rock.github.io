# 【Bedrock】でRAGチャットボットを作ってみた(超簡易版)

こんにちは。
最近ちょっとずつAIとクラウドに手を出し始めたエンジニアです。

今回はタイトルの通り、

現在の日本企業には、とても流行っているRagChatの超簡易版を作りました。


なぜ「超簡易版」と言いますか？

1.　ベクトルの設定（チャンク分割size、overlap）　→　しなかった

2.　プロンプトの工夫（プロンプトの組み立て）　→　しなかった

3.　Top-K(取得資料件数の設定)、rerank（検索で取得した資料、もう一回関連性の計算）　→　しなかった

4.　Pythonでゴリゴリ書く（つまり、Lambdaの書くのがない）　→　しなかった

5.　MetadataのFilter制御（例えば、平社員が財務情報を見えないので、答えのために、資料の範囲を制御）　→　しなかった

6.　guardrailの設定（敏感情報、わいせつ情報の遮断）　→　しなかった

7.　チャットのUI　→　しなかった

8.　性能の監視（CloudWatch）　→　しなかった

9.　監視、安全（Trail）　→　しなかった

10.　高可用性　→　しなかった

などなど....

もちろん、企業級RagChatbotの場合、上記のものはしっかり組立しないといけない、

今回の一旦細かい設定を置いておきます。

つまり、今回の実験は一旦にAWSのAI能力を楽しみたいです！


# 🎯 今回作ったもの

簡単に言うと：

> 社内規定や福利厚生を質問すると答えてくれるチャットボット

例えば：

```
最近ストレス解消したいですが、何か社内リソースを利用できますか？
```

👇 AIの回答：

* 社内カフェ（割引あり）
* 保養所の利用
* リラックス環境の説明

---

# 🧱 全体構成

今回の構成はこんな感じです：

```
ユーザー質問
   ↓

   ↓

S3内のドキュメント
   ↓
回答生成
```

# ちなみに、

もしKnowledge Baseの同期化が失敗する場合、policyの設定が問題がある可能性が高いです。

今までのAWS実操作の経験により、IAMとか、Roleとか、policyとかの権限設定は一番間違いやすいところです。

90%のチューニングは権限のチューニングです。

例えば、今回の実験は、最初やる時に、BedrockがKMSにアクセスの権限の付与は忘れました。
